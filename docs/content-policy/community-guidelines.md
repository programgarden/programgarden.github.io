# **D. 커뮤니티 콘텐츠 가이드라인**

**시행일:** 2026년 3월 12일

**요약:** 본 가이드라인은 ProgramGarden 커뮤니티에서 공유되는 DSL 프로젝트의 품질 기준과 금지 사항을 규정합니다. 건전한 커뮤니티 환경을 유지하기 위해 모든 사용자는 본 가이드라인을 준수해야 합니다.

---

## **1. 공유 DSL 품질 기준**

### 1. 코드 완결성
커뮤니티에 공유하는 DSL 프로젝트는 최소한의 완결성을 갖추어야 합니다. 의미 없는 코드, 빈 프로젝트, 테스트 목적의 임시 코드를 공유하는 것은 지양해 주시기 바랍니다.

### 2. 설명 제공
공유 프로젝트에는 프로젝트의 목적, 사용 방법 등 다른 사용자가 이해할 수 있는 최소한의 설명을 포함하는 것을 권장합니다.

### 3. AI 자동 태깅
공유된 프로젝트에는 AI가 자동으로 카테고리 태그를 생성합니다. 이를 통해 다른 사용자가 원하는 프로젝트를 쉽게 검색할 수 있습니다. 자동 태깅의 정확성은 보증되지 않으며, 사용자가 수동으로 태그를 수정할 수 있습니다.

---

## **2. 금지 콘텐츠**

### 1. 악성 코드
바이러스, 웜, 트로이 목마, 랜섬웨어, 크립토마이너 등 악성 코드를 포함하는 DSL 프로젝트의 공유는 엄격히 금지됩니다. 시스템 자원을 무단 점유하거나 다른 사용자의 실행 환경에 해를 끼칠 수 있는 코드도 포함됩니다.

### 2. 타인의 유료 전략 무단 공유
다른 사용자 또는 외부에서 유료로 판매되는 트레이딩 전략, 비공개 알고리즘을 무단으로 복제하여 공유하는 행위는 금지됩니다. 이는 저작권 침해에 해당할 수 있습니다.

### 3. 허위 성과 표시
실제와 다른 백테스트 결과, 수익률, 성과 데이터를 프로젝트 설명에 기재하여 타 사용자를 기만하는 행위는 금지됩니다.

### 4. 불법 목적의 프로젝트
시세 조종, 불법 거래, 사기 등 불법적인 목적으로 설계된 프로젝트의 공유는 금지됩니다.

### 5. 스팸 및 광고
의미 없는 콘텐츠를 대량으로 공유하거나, 외부 서비스/상품 홍보를 목적으로 커뮤니티를 악용하는 행위는 금지됩니다.

### 6. 유해 콘텐츠
폭력, 증오, 차별을 조장하거나, 개인정보를 노출하는 콘텐츠가 포함된 프로젝트의 공유는 금지됩니다.

---

## **3. 자격증명 보호**

### 1. 자동 정제
프로젝트 공유 시, API 키, 시크릿 키 등 자격증명 정보는 **자동으로 제거(정제)**됩니다.

### 2. 사용자 주의
자동 정제에도 불구하고, 사용자는 공유 전 프로젝트에 민감한 정보(비밀번호, 개인 정보, 독자적 전략의 핵심 로직 등)가 포함되어 있지 않은지 직접 확인할 책임이 있습니다.

---

## **4. 신고 및 조치 프로세스**

### 1. 신고 방법
본 가이드라인을 위반하는 콘텐츠를 발견한 경우, 서비스 내 신고 기능 또는 `support@programgarden.com`을 통해 신고해 주시기 바랍니다.

### 2. 신고 시 포함 사항
*   대상 프로젝트의 이름 또는 URL
*   위반 사유
*   (가능한 경우) 증빙 자료

### 3. 처리 절차
1.  신고 접수 후 **5영업일 이내** 검토를 진행합니다.
2.  가이드라인 위반이 확인되면 해당 콘텐츠를 삭제하거나 비공개 처리합니다.
3.  위반 사용자에게 경고 또는 제재 조치를 통보합니다.

### 4. 제재 단계
위반의 경중에 따라 다음 조치가 취해질 수 있습니다:
*   **1차:** 경고 및 콘텐츠 삭제
*   **2차:** 커뮤니티 공유 기능 일시 제한 (30일)
*   **3차:** 커뮤니티 공유 기능 영구 제한
*   **심각한 위반:** 계정 정지 또는 영구 해지
